Color usage (Tailwind-first)

Primary actions use bg-blue-600 with hover bg-blue-700

Secondary accents use blue-100 and blue-50

Page background uses bg-slate-50

Cards and surfaces use bg-white

Borders use border-slate-200

Primary text uses text-slate-800

Secondary text uses text-slate-500

Muted or helper text uses text-slate-400

Links use text-blue-600 with subtle hover states

Typography

Use a clean sans-serif system font (Inter-style)

Headings are medium to semibold, never overly bold

Body text is normal weight with comfortable spacing

Avoid uppercase text except for very small labels

Layout & spacing

Centered layouts with generous white space

Use px-6 / py-3 for inputs and buttons

Cards should have rounded-xl

Inputs and buttons should have rounded-lg

Keep consistent vertical rhythm using space-y-4 or space-y-6

Shadows & depth

Use subtle shadows only (shadow-md or shadow-lg)

Cards should appear slightly floating, never heavy

Buttons may have a soft shadow but not aggressive elevation

Forms & inputs

Inputs have light borders and white background

On focus, show a soft blue ring (focus:ring-2 focus:ring-blue-100)

Avoid dark or thick borders

Placeholder text should be muted and subtle

Buttons

Primary buttons are solid blue with rounded corners

Text is white, medium weight

Hover states are smooth and understated

No outlines or ghost styles unless necessary

Overall feel

Calm, clean, modern, and professional

Optimized for SaaS dashboards, auth pages, and enterprise tools

Visually similar to Stripe, Linear, Atlassian-style products

Design should feel “effortless” and not decorative-heavy
